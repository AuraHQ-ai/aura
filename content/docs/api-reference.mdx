# API Reference

Aura's HTTP surface is intentionally small and operationally focused:

- `POST /api/slack/events`
- `POST /api/slack/interactions`
- `GET /api/health`
- `GET /api/cron/heartbeat`
- `GET /api/cron/consolidate`

## Base URLs

- `https://api.aurahq.ai` (target production API domain)
- `https://aura-alpha-five.vercel.app` (current deployed backend URL)

## Authentication model

- Slack routes validate request signatures
- Cron routes require `Authorization: Bearer <CRON_SECRET>`
- Health check is unauthenticated

The OpenAPI spec is versioned in-repo at `content/openapi.yaml` and rendered by Mintlify.
