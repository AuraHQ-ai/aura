# Environment Variables

These variables define what I can access, how I reason, and which integrations I can operate safely.

## Required

- `DATABASE_URL`
- `SLACK_BOT_TOKEN`
- `SLACK_SIGNING_SECRET`
- `AURA_BOT_USER_ID`
- `AURA_ADMIN_USER_IDS`
- `CRON_SECRET`

## Recommended

- `SLACK_USER_TOKEN` (workspace search)
- `ANTHROPIC_API_KEY` (direct fallback path)
- `TAVILY_API_KEY` (web search)
- `E2B_API_KEY` (sandbox execution)
- `CURSOR_API_KEY` (cursor-agent automation)
- `CURSOR_WEBHOOK_SECRET` (validates cursor-agent webhook)

## Model settings

- `MODEL_MAIN`
- `MODEL_FAST`
- `MODEL_EMBEDDING`
- `MODEL_ESCALATION`

Runtime model settings can also be overridden from DB settings.

## Optional integrations

- `GOOGLE_EMAIL_CLIENT_ID`
- `GOOGLE_EMAIL_CLIENT_SECRET`
- `GOOGLE_BQ_CREDENTIALS`
- `ELEVENLABS_API_KEY`
- `TWILIO_ACCOUNT_SID`
- `TWILIO_AUTH_TOKEN`
- `TWILIO_PHONE_NUMBER`
- `COHERE_API_KEY`
- `BROWSERBASE_API_KEY`
- `BROWSERBASE_PROJECT_ID`

## Add variables with CLI

```bash
printf '%s' 'value' | npx --yes vercel@50.13.2 env add VAR_NAME production --scope realadvisor
```

Always use `printf '%s'` to avoid trailing newline issues.

After adding or changing variables, redeploy so I pick up the updated runtime environment.
