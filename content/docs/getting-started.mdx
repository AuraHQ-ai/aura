# Getting Started

I’m Aura, an autonomous AI teammate that lives inside your Slack workspace.

## Prerequisites

- A Slack workspace where you can install apps
- A Vercel account
- A PostgreSQL database with `pgvector` (Neon works well)
- AI provider credentials (for example `ANTHROPIC_API_KEY`)

## Quick start

### 1. Clone and install

```bash
git clone https://github.com/realadvisor/aura.git
cd aura
npm install
```

### 2. Configure environment variables

At minimum, I need:

- `DATABASE_URL`
- `SLACK_BOT_TOKEN`
- `SLACK_SIGNING_SECRET`
- `AURA_BOT_USER_ID`
- `AURA_ADMIN_USER_IDS`
- `CRON_SECRET`

See [Environment Variables](/deployment/environment-variables) for the full list.

### 3. Deploy

For this repo's operational setup, deploy with:

```bash
npx --yes vercel@50.13.2 --prod --scope realadvisor
```

### 4. Configure Slack app endpoints

Point your Slack app request URLs to my deployment:

- Events: `https://api.aurahq.ai/api/slack/events`
- Interactions: `https://api.aurahq.ai/api/slack/interactions`

If you use a different domain, replace `api.aurahq.ai` accordingly.

After changing scopes, reinstall the Slack app so I receive the updated permissions.

### 5. Verify health

```bash
curl -s https://api.aurahq.ai/api/health
```

I should answer with:

```json
{"ok":true,"timestamp":"<current-iso-time>"}
```

## First interaction test

Mention me in a Slack channel and ask:

> "Summarize what happened in this channel today."

If setup is correct, I’ll reply and store retrievable memory for later conversations.
