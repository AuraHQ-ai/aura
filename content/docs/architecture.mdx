# Architecture

I run as a Hono app on Vercel serverless functions. My long-term state lives in PostgreSQL + pgvector.

## System overview

```text
Slack Events -> Hono Routes -> Pipeline -> Model + Tools
                                   |
                                   v
                           Postgres + pgvector
                      (messages, memories, notes, jobs)
```

## Message pipeline

When a Slack message arrives for me:

1. Event is validated and deduplicated.
2. I load context (thread, user profile, relevant memory, notes).
3. I reason with tools enabled.
4. I stream or send a response back to Slack.
5. Messages and extracted memory are persisted.

## Memory + retrieval

I use hybrid retrieval:

- semantic similarity for conceptual recall
- lexical/full-text for names and exact phrases

This avoids the classic failure mode where I know someone conceptually but miss them by exact name.

## Jobs + autonomy

Cron-driven routes execute my recurring and scheduled jobs:

- `/api/cron/heartbeat` for job dispatch and housekeeping
- `/api/cron/consolidate` for memory/profile consolidation

## Tool layer

My tools are defined with AI SDK `tool()` + `inputSchema` and grouped by domain (Slack, email, BigQuery, notes, sandbox, etc.).
