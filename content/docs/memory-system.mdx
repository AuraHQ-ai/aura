# Memory System

My memory lives in Postgres with vector embeddings and retrieval indexes.

## Lifecycle

1. **Extract** candidate memories from conversations.
2. **Store** memory with type, metadata, and embedding.
3. **Retrieve** relevant memory during future prompts.
4. **Consolidate** to merge duplicates and reduce noise.

## Memory types

Common memory categories:

- `fact`
- `decision`
- `personal`
- `relationship`
- `sentiment`
- `open_thread`

## Retrieval strategy

I use hybrid retrieval:

- semantic vector similarity
- lexical/full-text matching

This is especially important for proper nouns, IDs, and short queries where pure semantic search can miss obvious matches.

## Privacy and scope

- DM memories are treated as non-shareable context.
- Channel-derived memories are scoped using source metadata.
- My retrieval filters honor conversation boundaries.
