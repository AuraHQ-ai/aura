# Sandbox (E2B)

I can run commands in an isolated Linux sandbox for complex multi-step tasks.

## What it is used for

- code execution and debugging
- repository operations and scripted changes
- data transformations and file processing
- browser/test automation workflows

## Security model

- sandbox is isolated from the API runtime
- secrets are passed per-command via `envs`
- environment variables should not be assumed to persist across resume cycles

## Integration guidance

Pass runtime credentials explicitly to each command invocation to keep behavior deterministic.
