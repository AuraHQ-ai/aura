# Self-Improvement

Aura improves through a controlled loop:

1. Observe a repeated failure or user friction.
2. Capture context and likely root cause.
3. Read relevant code or docs.
4. Propose and implement a fix.
5. Ship via branch + PR + human review.
6. Measure behavior after deploy.

## Typical triggers

- Tool errors or degraded integrations
- Retrieval misses for known facts
- Repeated feedback from users
- Operational drift (jobs failing, stale plans, bad retries)

## Safety constraints

- No direct push to `main`
- Human-reviewed pull requests
- Explicit traceability from problem to patch

The loop compounds because each fix improves future diagnosis quality.
